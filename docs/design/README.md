# Helios Design Review

This directory is the canonical place for reviewable product direction that is more concrete than a written requirement and less committed than production code.

- `prototypes/` holds clickable approval prototypes. They model intended form, information hierarchy and interaction states. They are not production code and must not be copied into the PWA without a design and implementation review.
- `assets/` holds the visual-reference index, asset provenance and the intake rules for Bradgate-specific materials.

The product principles and quality gate remain in the Personal Obsidian vault at `Helios - UI and Automation Spec.md`. Use this directory alongside that specification for all user-facing Helios work.

## Current review baseline

- `prototypes/helios-control-surface.html` covers the Home, Lounge and Energy compositions, including a confirmed command state and an unavailable camera state.
- `assets/visual-reference-index.md` records the established premium-control references and what each is useful for.
